Service Dependency Help

This page allows you to define Service Dependency objects within your Blue monitoring configuration. Service Dependencies are a way of suppressing notifcations and active checks of Services based on the status of another Service. When defining a Service Dependency there are a number of different pieces of information that you need to provide:

  • Dependent Host Name - This is the name of the Host on which the dependent Service runs.
  • Dependent Service Description - This is the description of Service on whose status we shall depend.
  • Host Name - The name of the Host on which the Service runs.
  • Service Description - The description of the Service that we would like to make dependent.

You can also specify whether or not this Dependency inherits the dependencies of the Service it is dependent on. Using this option, should any Services that the parent of this dependency is dependent on fail, then notifications and checks for this Service will also be supressed.

By providing the Execution and Notification failure criteria, you prescribe the conditions of the dependent Service that will cause checks and notifications of the Service to be supressed.

It should be noted that Service Dependencies are an optional part of your monitoring setup.
