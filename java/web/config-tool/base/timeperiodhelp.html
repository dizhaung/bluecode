Time Period Help

This page allows you to define Time Period objects within your Blue monitoring configuration. A Time Period object can be viewed as a definition of a period of time in which it is legal to perform Service and Host checks and send notifications.

When defining a Time Period object, there are two required pieces of information. The information you must supply is as follows:

  • Time Period Name - The name of this Time Period.
  • Alias - The alias of this Time Period.

There are two valid syntaxes for definining a Time Period. As shown on the Time Period form, there is a begin and end period for each day of the week. Each of these periods can be addressed in the following ways:

  • Begin:09:00 End17:00
  • Begin:09:00-12:00 End15:00-20:00

In the first example, the Time Period is valid from 09:00 through to 17:00 without any breaks. In the second example, the Time Period is valid from 09:00 until 12:00, and then from 15:00 until 20:00. This technique can be applied to each day of the week to create highly customised Time Periods. Should you not wish to have a valid Time on a certain day of the week, simply exclude that day from the Time Period definition.
