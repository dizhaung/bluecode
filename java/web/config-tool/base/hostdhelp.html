Host Dependency Help

This page allows you to define Host Dependency objects within your Blue monitoring configuration. Host Dependencies are a way of suppressing notifcations and active checks of Hosts based on the status of another Host. When defining a Host Dependency there are a number of different pieces of information that you need to provide:

  • Dependent Host Name - This is the name of the Host on which we are dependent..
  • Host Name - The host that this dependency applies to.

You can also specify whether or not this Dependency inherits the dependencies of the Host it is dependent on. Using this option, should any Hosts that the parent of this dependency is dependent on fail, then notifications and checks for this Host will also be supressed.

By providing the Execution and Notification failure criteria, you prescribe the conditions of the dependent Host that will cause checks and notifications of the Host to be supressed.

It should be noted that Host Escalations are an optional part of your monitoring setup.
